# Canonical Minimal DSR Package Example

This directory contains a synthetic worked example for exercising the DSR Framework templates, schemas, checklists, contribution logic, and transparency expectations.

The example project is the **Neighborhood Heat-Risk Cooling Route Advisor**, a fictional decision-support method and lightweight prototype specification for helping community health workers prioritize outreach routes during heat-risk events.

## Example status

This example is demonstration material for the DSR Framework. It is not empirical evidence that the DSR Framework is effective, and it is not evidence that the fictional cooling-route artifact is safe or effective in real public-health operations.

## Files

| File | Role |
|---|---|
| `project-charter.md` | Project identity, scope, stakeholders, roles, and non-goals. |
| `problem-and-objectives.md` | Problem space, objectives, requirements, and success criteria. |
| `artifact-specification.md` | Artifact boundary, type, components, inputs, outputs, and use context. |
| `design-rationale.md` | Design choices, alternatives, risks, and rationale. |
| `evaluation-plan.yaml` | Structured evaluation plan exercising the evaluation schema/template. |
| `contribution-claim.yaml` | Structured contribution claim exercising contribution positioning. |
| `transparency-crosswalk.yaml` | Process/problem/solution/build/evaluation/contribution transparency record. |
| `package-inventory.yaml` | Example-local file inventory. |
| `validation-notes.md` | Notes on what this example exercises and what it does not prove. |

## Intended use

Use this example to check whether a researcher can follow the framework from problem framing through artifact specification, evaluation planning, contribution positioning, and transparency reporting.

## Non-claims

This example does not claim:

- real-world deployment;
- clinical or public-health effectiveness;
- generalizable heat-risk prediction validity;
- external reuse of the DSR Framework;
- L5 archival/publication-ready package status.
